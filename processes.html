LIFECYCLE The Chronological Flow

The 7 Processes

Processes describe a step-by-step progression from the pre-project start to the final closure. They tell you "What to do, When, and by Whom."

Overview Principles Practices Roles Processes Documents Glossary

Process Model Overview

SU
Pre-Project

Starting Up a Project

Мета: Перевірити, чи є проєкт життєздатним ("Do we have a viable project?").

  • Призначення Executive та PM
  • Створення Project Brief
  • План стадії ініціації
DP
Project Board

Directing a Project

Єдиний процес для **Ради Проєкту**. Триває від SU до CP.

  • Затвердження ініціації та кожного етапу
  • Прийняття рішень по Exceptions
  • Надання спеціальних порад PM
IP
Initiation Stage

Initiating a Project

Створення міцного фундаменту та детального планування.

  • Формування PID
  • Детальний Business Case
  • Стратегії управління (Ризики, Якість)
CS
Delivery Stages

Controlling a Stage

Повсякденна робота PM з управління робочою стадією.

  • Авторизація Work Packages
  • Аналіз звітів (Checkpoint Reports)
  • Управління Issue та Risk
MP
Team Manager

Managing Product Delivery

Процес створення технічних продуктів проєкту.

  • Прийняття та виконання Work Package
  • Демонстрація якості (Quality Records)
  • Передача готових продуктів PM
SB
Boundary Point

Managing a Stage Boundary

Підготовка до наступного етапу та звіт за поточний.

  • План наступної стадії
  • Оновлення Project Plan та Business Case
  • Звіт End Stage Report
CP

Closing a Project

Final Stage

Організоване завершення проєкту. Запобігає безкінечному "дрейфу".

  • Передача продуктів
  • Звіт про витягнуті уроки
  • Звіт про завершення проєкту
  • Рекомендація про закриття
Mastering the flow: SU → IP → CS/MP (Loop) → SB → CP
