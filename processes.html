SU
Pre-Project

Starting Up a Project

Ensures that the prerequisites for Initiating a Project are in place. Is the project viable?

  • Appoint Executive and Project Manager
  • Create the Project Brief (The "Outline")
  • Define the Project Product Description
DP
Project Board

Directing a Project

The only process for the Project Board. It runs from startup until the final closure.

  • Authorize Initiation and Delivery Stages
  • Manage by Exception (Only act if tolerances are exceeded)
  • Provide ad-hoc direction and guidance
IP
Initiation Stage

Initiating a Project

Establishes the firm foundations and detailed plans for the project.

  • Assemble the PID (Project Initiation Documentation)
  • Setup Project Quality and Risk Approaches
  • Refine the Business Case into its full version
CS
Project Manager

Controlling a Stage

The Project Manager’s day-to-day management of a delivery stage.

  • Authorize and monitor Work Packages
  • Capture and analyze issues and risks
  • Report progress via Checkpoint Reports
MP
Team Manager

Managing Product Delivery

The link between the Project Manager and the people building the products.

  • Accept and execute Work Packages
  • Carry out the quality checks
  • Hand over completed products to the PM
SB
Boundary Point

Managing a Stage Boundary

Preparing the Board to make a "Go/No-Go" decision on the next stage.

  • Create the Next Stage Plan
  • Update the Business Case with current costs
  • Produce the End Stage Report
CP

Closing a Project

Final Stage Only

Provides a fixed point to check that products are accepted and the project can end.

  • Hand over products formally
  • Capture Lessons Learned
  • Issue End Project Report
  • Notify corporate that project is ending
Mastering the flow: SU → IP → CS/MP (Loop) → SB → CP
