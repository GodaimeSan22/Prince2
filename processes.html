LIFECYCLE The Chronological Flow

The 7 Processes

Processes describe a step-by-step progression from pre-project start to final closure. They define "What to do, When, and by Whom."

Overview Principles Practices Roles Processes Documents Glossary

Process Model Overview

[Visual Diagram: SU → IP → Cycle (CS & MP) → SB → CP | Overseen by DP]
SU
Pre-Project

Starting Up a Project

Purpose: To ensure that the prerequisites for Initiating a Project are in place. ("Do we have a viable project?")

  • Appoint Executive and Project Manager
  • Create the Project Brief
  • Plan the Initiation Stage
DP
Project Board

Directing a Project

The only process for the **Project Board**. It runs from startup until closure.

  • Authorize initiation and each delivery stage
  • Make decisions on Exceptions
  • Provide ad-hoc direction and guidance
IP
Initiation Stage

Initiating a Project

Building a solid foundation and detailed planning for the project.

  • Assemble the PID (Project Initiation Documentation)
  • Refine the Business Case
  • Setup Management Strategies (Risk, Quality, etc.)
CS
Delivery Stages

Controlling a Stage

The Project Manager’s day-to-day management of a delivery stage.

  • Authorize Work Packages
  • Review stage status (Checkpoint Reports)
  • Manage Issues and Risks
MP
Team Manager

Managing Product Delivery

The process of creating the project’s technical products.

  • Accept and execute Work Packages
  • Demonstrate quality (Quality Records)
  • Hand over completed products to the PM
SB
Boundary Point

Managing a Stage Boundary

Preparing for the next stage and reporting on the current one.

  • Create the Next Stage Plan
  • Update Project Plan and Business Case
  • Produce the End Stage Report
CP

Closing a Project

Final Stage

An organized finish to the project. Prevents "project drift" and ensures formal handover.

  • Hand over products
  • Capture Lessons Learned
  • Produce End Project Report
  • Recommend Project Closure
Mastering the flow: SU → IP → CS/MP (Loop) → SB → CP
