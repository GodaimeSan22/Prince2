ORGANIZING Project Management Team

Roles & Responsibilities

A PRINCE2 project has defined roles. It is not about "job titles," but about "responsibilities." One person can hold multiple roles, or one role can be shared.

Overview Principles Practices Roles Documents Glossary

Executive

THE OWNER

Primary Focus: Ultimately accountable for the project's success. Ensures the project provides value for money.

Reports To:

Corporate, Program Management, or the Customer.

Key Responsibility:

Owns the Business Case. Appoints the PM and the rest of the Board.

Can it be combined? Usually one individual. Cannot be the Project Manager or Project Assurance.

Project Manager

THE DRIVER

Primary Focus: Day-to-day management of the project. Managing within the constraints (tolerances) set by the Board.

Reports To:

The Project Board (specifically the Executive).

Key Responsibility:

Planning, monitoring, and controlling. Escalates exceptions.

Can it be combined? Yes. Can act as Team Manager or Project Support (on small projects). Cannot be Project Assurance.

Senior User

THE BENEFICIARY

Primary Focus: Represents the people who will use the products. Defines the requirements and monitors benefits.

Key Responsibility:

Provides the User resources. Responsible for the quality of products (from a user perspective).

Can it be combined? Yes. The Executive can also be the Senior User (especially in small projects).

Senior Supplier

THE BUILDER

Primary Focus: Represents the specialists who design, develop, and deliver the project's products.

Key Responsibility:

Provides technical integrity and supplier resources. Accountability for the quality of products (technical).

Project Assurance

THE AUDITOR

Primary Focus: To provide the Board with independent confidence that the project is being managed correctly.

EXAM TIP:

Assurance MUST be independent of the Project Manager. The Board members usually do this themselves, or they can delegate it.

Can it be combined? NEVER with the Project Manager or Team Manager roles.
